# JoyPhone 海选答辩 · 图张大纲（PPT 分镜）

> 评委只扫一眼图就该懂，文字≤15字/页。下方给出每页的画面、口播锚点、以及可直接喂给 AI 绘图的 Prompt。

## 页 3 · 手-眼-脑 架构图（核心）★

- **画面**：上下两层，中间 WebSocket 双向箭头。
  - 上层「云端大脑 · FastAPI」：决策引擎 / 协商机器人 / 技能库 / LLM抽象层。
  - 下层「安卓工作手机 · 手和眼」：**眼=无障碍读屏(节点树+截图)**，**手=无障碍执行(点击/输入/滑动)**。
  - 下层底部一条横幅：**vivo 企业 SDK：强制开启无障碍·用户关不掉·应用卸不掉（合规底座）**。

- **一句话标题**：**手机是手和眼，云端是大脑**
- **口播锚点**：讲稿"方案与创意"段 `[00:40]→[01:20]`——这页要停最久
- **说明**：本页同时回答评委必问的"无障碍 vs SDK 角色"——无障碍在下层做手眼，SDK 是横幅底座。参见 [`答辩-03-预演QA.md`](答辩-03-预演QA.md) Q1。

## Demo 闭环（3分钟问答时播放的视频/连续图，非讲解正片）

> 用连续截图或录屏，展示一条**完整闭环**，让评委"看到它真的在动"：

1. 云端后台创建任务：给"张三"发协商消息，目标=确认还款时间。
2. 手机自动打开飞书 → 搜索联系人 → 进入会话（画面高亮无障碍识别到的"搜索框/联系人"）。
3. 输入首轮话术 → **发送前二次确认**（强调安全：错人绝不发）→ 发送成功。
4. 对方回复 → 云端理解 → 自动生成第二轮协商话术 → 再发送。
5. 命中已沉淀技能时，右上角标"技能命中·秒执行"，对比首次的"大模型思考中"，直观体现越用越快。
6. 收尾字幕：整个过程**零人工、可回放、话术合规**。

**Demo 叙事关键帧建议做成 3-5 张连续图**，即使没时间放视频也能翻图讲完。

## 制图注意事项

- 全套配色统一：蓝→紫渐变（科技感），红色只用于"痛点/风险"。
- 架构图（页3）和飞轮（页4）是**打分核心页**，制作精度优先级最高。
- 每页右下角放小页码，方便 Q&A 时说"请翻回第 3 页"。
- 不要在正片里堆代码/接口名；技术细节放备用附录页，被追问再翻。
