# JoyPhone 海选答辩 · 预演 Q&A（3分钟提问环节）

> 目标：评委追问时**10秒内切入要点**，先给结论再补一句支撑。
> 全场纪律：讲愿景不纠结实现度；不用"外挂/爬虫/破解"等敏感词；无障碍与 SDK 分开讲；只报三数字。
> 每题给「一句话答」+「可展开」+「翻哪页图」。

---

## Q1 · 无障碍和 vivo SDK 各扮演什么角色？（必问）★

- **一句话答**：无障碍是"手和眼"，负责像人一样读屏和操作任意 APP；vivo 企业 SDK 是"合规底座"，负责把这只手焊死在工作手机上。
- **可展开**：无障碍读节点树+截图感知界面、执行点击输入滑动，一套通吃飞书/企微/短信，不依赖厂商配合；vivo SDK 通过 MDM 能**强制开启无障碍且用户关不掉**、**应用不可卸载常驻保活**，保证企业场景下稳定合规。
- **翻页**：图张大纲 页3（架构图底部横幅）。

## Q2 · 你们现在做到哪一步了？实现度如何？（尖锐）★

- **一句话答**：核心的"手眼脑"闭环链路我们已经跑通验证，本次海选重点是把这套**创意和演进蓝图**讲清楚。
- **可展开**：云端决策、无障碍执行、协商闭环的关键环节都已打通；vivo 企业 SDK 的合规能力是我们的下一步集成规划，SDK 文档能力已对齐。
- **注意**：不要主动说"SDK 还没集成"，被追问时说"合规底座是规划中的下一步，能力已对齐文档"。

## Q3 · 和市面上的短信/外呼/群控/RPA 有什么不一样？

- **一句话答**：别人认死坐标、绑单一通道、越用越僵；我们**像人一样看懂界面、一套通吃所有社交APP、越用越聪明**。
- **可展开**：语义识别界面所以改版不崩；真机真人操作所以抗风控；成功路径自动沉淀成技能，形成抄不走的数据护城河。
- **翻页**：图张大纲 页5 对比表。

## Q4 · 这会不会有合规/法务风险？（必问）★

- **一句话答**：我们定位是**企业自有工作手机 + 员工授权场景**，不碰个人隐私、不做未授权操作，话术 100% 由云端合规模板控制。
- **可展开**：① 设备是企业配发、员工知情授权；② 所有触达话术走审核过的模板，可回放可审计；③ 发送前二次确认，绝不发错人；④ vivo 企业 SDK 提供防篡改和白名单联网，进一步收紧边界。
- **底线话术**：合规是产品的**前提而非补丁**，我们从设计上就把它做进了闭环。

## Q5 · 无障碍会不会被系统/APP 限制、被判定异常？

- **一句话答**：这正是我们引入 **vivo 企业 SDK 做底座**的原因——企业级 MDM 保证无障碍稳定开启不被关闭。
- **可展开**：面向企业工作手机场景，而非个人机破解；配合真机真人节奏操作，降低被判异常概率；界面识别用语义锚点而非坐标，改版自适应。

## Q6 · "越用越聪明"具体怎么实现？会不会只是噱头？

- **一句话答**：每条成功执行路径会被抽象成可复用"技能"存进技能库，下次同类任务**直接命中秒执行**，不用再让大模型现想。
- **可展开**：命中技能=快且省 LLM 成本；未命中才回退大模型现学并沉淀新技能；用得越多技能库越厚、越懂业务，这就是数据护城河。
- **翻页**：图张大纲 页4 飞轮。

## Q7 · 目标用户是谁？谁会为它买单？

- **一句话答**：需要**大规模合规触达客户**的企业——银行/消金的客服与催收、运营商与电商的营销通知。
- **可展开**：他们的痛点是外呼短信被风控、社交APP没API、人工坐席贵且话术不统一；我们用一部工作手机把这三件事一起解决。
- **画像**：一线业务坐席从"手动逐个操作"变成"审核与兜底"，人效大幅提升。

## Q8 · 可能的技术漏洞/失败场景怎么兜底？（体现工程严谨）

- **一句话答**：我们对每类失败都设计了 fail-closed 兜底，宁可不做也不做错。
- **可展开**（点到即止，各一句）：
  - 界面变了坐标失效 → 语义锚点识别，识别不到就停，不瞎点。
  - 操作卡住/回调延迟 → 不等回调、超时熔断，不堵队列。
  - 技能重放异常 → 熔断降级回大模型。
  - 发送对象核对 → 发送前二次确认，错人绝不发。

## Q9 · 为什么绑定 vivo？换手机怎么办？

- **一句话答**：无障碍"手眼"是**安卓通用能力，不绑厂商**；vivo SDK 只是我们优先落地的企业合规底座，架构上可平滑扩展到其他 MDM。
- **可展开**：核心竞争力在云端大脑+技能库+通用无障碍，厂商底座是可替换的一层。

## Q10 · 2分钟没讲到的技术细节（决策引擎/端云协同）被追问

- **一句话答**：云端有分级决策引擎——**能用技能就用技能，不行才升级到大模型**，端云用 WebSocket 双向长连实时协同。
- **可展开**：分级回退保证又快又省；长连接保证手机执行结果实时回传、云端实时下发下一步指令。

## 抢答与控场小抄

- 每个回答**先给结论句**，再决定要不要展开；评委点头就停。
- 被问到不确定的实现细节：`"这块是我们下一阶段的重点，方案上已经想清楚：……"` 然后拉回创意。
- 时间紧时优先答 Q1/Q2/Q4（角色、实现度、合规）——这三题几乎必问。
- 全程不说：外挂、爬虫、破解、绕过、模拟点击（说"像人一样操作/无障碍执行"）。
